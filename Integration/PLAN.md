# Stockpile Detection & Volume Estimation Pipeline - Implementation Plan

## Overview
Automated headless Python pipeline that takes ODM photogrammetry outputs, detects stockpiles using SAM (Segment Anything Model), estimates volumes using 3 methods, and generates PDF + JSON reports.

## Input Data Summary
- **Orthophoto**: `odm_orthophoto.tif` — 130x156 px, 4-band RGBA, 5cm/px, EPSG:32631
- **DSM**: `dsm.tif` — 129x155 px, float32, 5cm/px, EPSG:32631
- **Point cloud**: `odm_georeferenced_model.laz` — 4.7M points
- **Bounds**: `odm_georeferenced_model.bounds.geojson` — convex hull polygon
- **GPU**: NVIDIA RTX 4060 (CUDA available)

## Module Details

### Module 1: `loader.py` — Data Loader
- Load orthophoto as rasterio dataset (RGBA GeoTIFF)
- Load DSM as rasterio dataset (float32 GeoTIFF)
- Load point cloud using laspy (LAZ → numpy arrays of X, Y, Z)
- Load bounds GeoJSON
- Validate CRS consistency (all must be EPSG:32631)
- Extract metadata: resolution, extents, band info
- Return a `PipelineData` dataclass with all loaded data

### Module 2: `segmentation.py` — SAM Segmentation
**Using `segment-geospatial` (samgeo):**
1. Initialize `SamGeo` with `sam_vit_h` model checkpoint (GPU-accelerated)
2. Run `generate()` on the orthophoto GeoTIFF for automatic mask generation
3. Parameters: `min_mask_region_area=100` to filter tiny segments
4. Output: segmentation mask as GeoTIFF (each object has unique ID)
5. Convert to vector polygons via `tiff_to_geojson()`

**Alternative: Geo-SAM (configurable)**
- Same interface, different backend
- Config flag to switch between approaches

### Module 3: `stockpile_detector.py` — Stockpile Classification
From the segmented polygons, identify which are stockpiles:
1. **Area filter**: Keep segments within expected stockpile size range (configurable min/max m²)
2. **Elevation profile**: For each candidate polygon, sample DSM values inside it
   - Stockpiles have elevated centers relative to edges (convex elevation profile)
   - Calculate: mean_interior_elevation - mean_edge_elevation > threshold
3. **Shape filter**: Compactness ratio (area / perimeter²) — stockpiles tend to be roughly circular/oval
4. **Output**: List of stockpile polygons with confidence scores + classified GeoJSON

### Module 4: `volume.py` — Volume Estimation (3 Methods)
For each detected stockpile polygon:

**Method A: DSM - Base Plane**
- Extract DSM elevations at polygon boundary vertices
- Fit a least-squares plane through boundary points: z = m1*x + m2*y + b
- Clip DSM to polygon extent
- Volume = Σ(DSM - plane) × pixel_area × to_meters³
- Adapted from WebODM's `calc_volume()` with `base_method="plane"`

**Method B: DSM - DTM (Interpolated Ground)**
- Since no DTM was generated by ODM, generate a pseudo-DTM:
  - Sample ground-level elevations from polygon boundary
  - Use Delaunay triangulation + linear interpolation (`griddata`) to create ground surface
  - This is equivalent to WebODM's `base_method="triangulate"`
- Volume = Σ(DSM - interpolated_ground) × pixel_area × to_meters³

**Method C: Point Cloud Direct**
- Extract points within stockpile polygon from LAZ
- Separate ground points (boundary region) from stockpile points
- Fit a base plane through ground points
- Compute volume using 2.5D integration: for each XY cell, height = Z_point - Z_base
- Sum over all cells × cell_area

All three methods return volume in cubic meters.

### Module 5: `report.py` — Report Generation
**PDF Report (using reportlab):**
- Page 1: Title, survey metadata (date, location, CRS, area, GSD)
- Page 2: Orthophoto overview with stockpile boundaries overlaid
- Page 3: Segmentation results — colored mask overlay
- Page 4: Per-stockpile detail — cropped image, boundary, elevation profile
- Page 5: Volume results table (all 3 methods per stockpile), comparison chart

### Module 6: `config.py` — Configuration
```python
@dataclass
class PipelineConfig:
    # Input paths
    orthophoto_path: str
    dsm_path: str
    pointcloud_path: str
    bounds_path: str
    output_dir: str

    # SAM settings
    sam_model: str = "vit_h"          # vit_h, vit_l, vit_b
    sam_backend: str = "samgeo"       # "samgeo" or "geosam"
    device: str = "cuda"              # "cuda" or "cpu"

    # Stockpile detection thresholds
    min_stockpile_area_m2: float = 0.5
    max_stockpile_area_m2: float = 50.0
    elevation_threshold_m: float = 0.3
    compactness_threshold: float = 0.1

    # Volume methods to run
    volume_methods: list = ("plane", "triangulate", "pointcloud")
```
